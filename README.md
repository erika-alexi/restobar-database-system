# Restobar Database System

## Overview

This project is a relational database system designed to support daily operations of a restobar. It manages customer transactions, staff assignments, menu items, inventory, and supplier data using a structured and normalized database design.

## Key Features

* Customer information management
* Order processing with detailed transaction records
* Staff tracking for order handling
* Inventory and supplier management system
* Menu categorization and pricing control

## Database Design

The system is built using a well-structured Entity-Relationship Diagram (ERD) with the following entities:

* Customers
* Staff
* Orders
* OrderDetails
* Suppliers
* Stocks
* MenuItems
* MenuCategory

Relationships were carefully designed to maintain referential integrity and support efficient data retrieval.

## Technical Highlights

* Implementation of primary and foreign keys
* One-to-many relationships across core entities
* Normalized database structure to reduce redundancy
* Separation of transactional and master data

## Technologies Used

* Microsoft Access (.accdb)
* Lucidchart for Entity-Relationship Modeling (ERD)

## Project Structure

* `/ERD` – Contains the ERD diagram (PDF)
* `/Database` – Contains the Microsoft Access database file
* `/Documentation` – Contains proof of tables, relationships, and sample data

## How to Run

1. Download the database file from the `/Database` folder
2. Open using Microsoft Access
3. Navigate through tables and relationships to explore the system
